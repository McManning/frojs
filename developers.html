Configuration

fro.World accepts a number of configuration options for each module (network, renderer, camera, etc). The following is a quick description of each configurable module in detail, followed by a complete example of available options.

Camera

Controls default camera behavior.

Option Default Type Description
bounds null array of 4 integers Bounding box that the camera will always stay within (in pixels) [top, left, right, bottom]. Note that this must be larger than the canvas size
trackPlayer true boolean Whether the camera should automatically follow our Actor as it moves. If this option is true, the position option is ignored. Manually calling setCenter() or trackEntity() on the camera will cancel out the effects of this setting
position null array of 2 integers [x, y] position of the camera, in pixels. If trackPlayer is true, this option is ignored

Network

Options for connecting to a Frojs-compliant server for multiplayer functionality. For networking to work, socket.io must be available at the global scope (specifically, window.io)

Option Default Type Description
server string Url that points to a Frojs-compliant server.
token null string Value that can be used to verify the client's identity on the server after connection. The way this is used depends on the specific server implementation
room null string Room to join after successful authentication. This allows servers to support multiple client groups simultaneously on one instance. Note that the actual room the client is put in is up to the specific server implementation, and may differ from the original request

Player

Configurations specific to the user's own entity instance. This includes setting the avatar, default position/state, display name, etc.

Option Default Type Description
template null string ID of the template to inherit for the Actor's configuration. If not specified, you must define all required properties of the client's Actor here.

For additional options specific to configuring the client's Actor (such as avatar and position), check out the Actors section of the Creating Worlds guide.

Renderer

Controls the actual display of content and which features or optimizations to toggle.

Option Default Type Description
canvas HTMLCanvasElement <canvas> element for rendering Frojs content
usesWebGL true boolean Whether the client will try to use WebGL when available
background [0, 0, 0] array of 3 integers Canvas background color, as RGB [0-255]

World

Configuration for the loaded world itself, including entity definitions and interactions.

For a complete guide on how to create this world configuration, visit the Creating Worlds guide.

Option Default Type Description
templates null array of objects Templates that define default options that can be shared among different entity instances
entities null array of objects Entity instances that will be loaded with the world including props, actors, sounds, etc

Configuration Example

Complete fro.World configuration example: 


{
    camera: {
        bounds: [-800, -600, 800, 600],
        trackPlayer: true,
        position: null // Using trackPlayer instead
    },
    network: {
        server: 'http://universe.frojs.com',
        token: 'MySecretToken',
        room: 'examples'
    },
    player: {
        avatar: {
            type: 'Animation',
            url: "http://i.imgur.com/MAT9aD2.png",
            autoplay: true,
            width: 32,
            height: 64,
            keyframes: {
                move_2: {
                    loop: true,
                    frames: [0, 1000, 1, 1000]
                },
                move_8: {
                    loop: true,
                    frames: [3, 1000, 4, 1000]
                },
                move_4: {
                    loop: true,
                    frames: [6, 1000, 7, 1000]
                },
                move_6: {
                    loop: true,
                    frames: [9, 1000, 10, 1000]
                },
            }
        },
        position: [0, 0, 0],
        name: 'Example Player',
        direction: 2, // facing south
        action: 0 // idle
    },
    plugins: {},
    renderer: {
        canvas: document.getElementById('fro-canvas'),
        background: [0, 0, 0]
    },
    world: {
        templates: [
            {
                id: 'crate',
                type: 'Prop',
                image: {
                    type: 'Image',
                    url: 'http://i.imgur.com/2LlSRc8.png',
                    fitToTexture: false,
                    width: 132,
                    height: 143
                },
                offset: [0, -24],
                collisions: [-66, -95, 132, 98]
            }
        ],
        entities: [
            {
                template: 'crate',
                position: [0, 0, 0]
            }
        ]
    }
}
